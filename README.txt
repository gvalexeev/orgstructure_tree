Древовидное отображение оргштатной структуры:
Сделано:
- создание пользователей и департаментов. 
При нажатии на кнопку "Создать", появляется модальное окошко, в котором можно выбрать тип создаваемого объекта - пользователь или департамент.
При выборе одного из вариантов селектора, список выбора скрывается и появляется форма, с нужными полями. На поля "Департамент" и "Родительский департамент" навешаны автокомплиты. 

- удаление узлов дерева
При выделении узла появляется кнопка "Удалить".
Возможно удалить только те узлы, которые не содержат дочерних узлов. При попытке удаления другого узла, возникает модальное окно, с предупреждением что данный узел содержит дочерние.


Не сделано(увы, тут список больше):
- поиск
Задуманный принцип действия:
При нажатии на кнопку "Поиск", происходит запрос на сервер. Сервер возвращает нам следующую json структуру

{
	id,
	level,
	path[]
}

в которой path[] - это отображение пути до найденного элемента относительно корневой ноды в виде массива id. После, на клиенте запускаем два вложенных цикла, один по найденным объектам, другой по элементу path[] и раскрываем каждую найденную ноду вплоть до конечного элемента(и подсвечиваем красным). Но, так как я руководствовался соображением, что хранить полную структуру на клиенте - не гуд, то возникает проблема динамической подгрузки нод. При подгрузке ноды, отправляется запрос на сервер, который выполняется асинхронно и мы не получаем данные о дочерних элементах ноды вовремя. В этом и была проблема...  

Что хотелось бы сделать:
- сохранение вида дерева в кукисах или localstorage
- отверстать(иконки кнопкам, красивое отображение дерева)
